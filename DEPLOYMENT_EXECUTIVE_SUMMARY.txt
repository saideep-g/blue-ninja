================================================================================
  ANALYTICS ENRICHMENT (CRITICAL SERVICE)
================================================================================

WHAT HAPPENS:
  1. Student submits answer
  2. Create v1 log (existing process)
  3. Validate v1 fields (STRICT)
  4. Look up atom in v2 curriculum
  5. Enrich log with:
     - atomIdV2 (CBSE7.CH01.INT.01 format)
     - templateId (NUMERIC_INPUT, MCQ_CONCEPT, etc.)
     - moduleId (CBSE7-CH01-INTEGERS)
     - domain (Integers, Algebra, etc.)
     - masteryProfileId (MP_CORE_FLUENCY, etc.)
     - learningBehavior (FLUENCY, BUILDING, STRUGGLING, etc.)
     - outcomeIds (learning objectives)
     - And 10+ more optional fields
  6. Validate v2 fields (PERMISSIVE)
  7. Save enriched log to Firestore
  8. Continue with normal flow

WHY IT MATTERS:
  - Teachers get insights into what, not just accuracy
  - Parents see progress by chapter/module
  - Admins can analyze by template/domain/behavior
  - Students get personalized recovery
  - Platform learns optimal question sequencing

SAFETY:
  - v1 validation must pass (strict)
  - v2 enrichment optional (graceful fallback)
  - If curriculum lookup fails, log still saved
  - If enrichment fails, warning logged but continues
  - No breaking changes to any existing systems

================================================================================
  DAILY MISSION STRUCTURE: 14+ SLOTS
================================================================================

SLOT 1-3: WARM-UP (Spaced Review)
  Q1: NUMERIC_INPUT     - Fast fact retrieval
  Q2: MCQ_CONCEPT       - Conceptual understanding
  Q3: MATCHING          - Visual representation
  Why: Activate prior knowledge

SLOT 4-6: DIAGNOSIS (Understanding Check)
  Q4: MCQ_CONCEPT       - Probe misconceptions
  Q5: ERROR_ANALYSIS    - Detect common errors
  Q6: NUMBER_LINE_PLACE - Test representation
  Why: Identify gaps before practice

SLOT 7-9: GUIDED PRACTICE (Interactive Learning)
  Q7: BALANCE_OPS/SLIDER - Procedural modeling
  Q8: CLASSIFY_SORT      - Classification skills
  Q9: DRAG_DROP_MATCH    - Correspondence
  Why: Build procedural fluency

SLOT 10-12: ADVANCED (Deep Thinking)
  Q10: STEP_BUILDER      - Multi-step reasoning
  Q11: MULTI_STEP_WORD   - Real-world application
  Q12: EXPRESSION_INPUT  - Symbolic fluency
  Why: Develop problem-solving

SLOT 13-14: REFLECTION (Consolidation)
  Q13: SHORT_EXPLAIN     - Metacognitive justification
  Q14: TRANSFER_MINI     - Novel context application
  Why: Solidify learning and prepare for transfer

ADAPTIVE SELECTION:
  Warm-Up:       Atoms not seen recently (spaced review)
  Diagnosis:     Atoms where student struggles (target weaknesses)
  Practice:      Mix of weak and strong (balanced progression)
  Advanced:      Progressive difficulty (stretch goals)
  Reflection:    Varied atoms (prepare for transfer)
